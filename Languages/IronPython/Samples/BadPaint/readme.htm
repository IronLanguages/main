Bad Paint

Bad paint is a sample paint program built upon the Windows Presentation Foundation framework.  What distinguishes this paint application from others is it's use of Microsoft .NET 4.0's Dynamic Language Runtime (DLR) feature.  The DLR is utilized to dynamically and programmatically modify visual objects within the paint canvas.  Such a feat is extremely difficult to accomplish with statically typed languages, and the DLR makes this a breeze to implement for Python.

 

Prerequisites

The prerequisites to successfully run the Bad Paint demo are:

  • Microsoft .NET Framework Version 4.0
    • Download from here
  • IronPython 2.6.1 for .NET 4.0
    • If you are new to IronPython, please go over the tutorial that comes with this distribution
    • Download from here
  • IronRuby 1.0v4
    • Optional.  Available from here

Getting Started

Please make sure the sample folder, BadPaint, (where this readme.htm is located) contains at least the following:

.\App.config
.\App.xaml
.\App.xaml.cs
.\BadPaint.csproj
.\BadPaint.sln
.\demo_snippets.py
.\MainWindow.xaml
.\MainWindow.xaml.cs
.\Properties\

                    AssemblyInfo.cs
                   Resources.Designer.cs
                   Resources.resx
                   Settings.Designer.cs
                   Settings.settings


IMPORTANT NOTE: if the folder containing the IronPython distribution is not in the default MSI install location, "%ProgramFiles%\IronPython 2.6 for .NET 4.0", you'll need to modify BadPaint.csproj to point to the correct installation location.  This means you'll need to adjust the HintPath elements within BadPaint.csproj which can be done either with notepad.exe or Visual Studio 2010.

Building Bad Paint

We recommend building the "Release" configuration of  BadPaint.sln directly from Visual Studio, although you can also build the demo from the command prompt by navigating to the BadPaint directory and typing the following:

C:\Windows\Microsoft.NET\Framework\v4.0.30319\MSBuild BadPaint.sln /p:configuration=Release

 

Running this command or building directly from Visual Studio will yield bin\Release\Demo.exe.

Playing with Demo.exe

From the command prompt, navigate to the BadPaint directory and type the following:

> bin\Release\Demo.exe

 

An application similar to the screenshot below will then start:

 

Try dragging a square onto the black canvas and follow this up by dragging the color green into the square:

 

At this point let's try to dynamically modify the canvas.  First, click on the green square to select it.  Now type "A" into the "Selected Item" textbox and strike the Enter key once.  Behind the scenes, this sequence of actions has associated our green square as the "A" attribute of the Application object from IronPython. 

Now click the "Language" drop-down menu and select the IronPython 2.6 option.  This allows us to input some Python code into the textbox immediately below the "Language" drop-down menu.  Type:

Application.A.Width = 100

into this box:

 

Finally, click the "Run Code" button to actually run your Python code against the canvas contents:

As you can imagine, at this point we've only touched the surface so to speak with respect to what's possible from the scripting code textbox.  This sample includes a demo_snippets.py file which as its name might imply, includes a few less trivial snippets of Python code intended to be copied and pasted into this sample's textbox and run.

This concludes our tutorial. Have fun playing with this sample!
