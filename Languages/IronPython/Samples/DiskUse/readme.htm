DiskUse - Graphical Disk Usage Utility

Contributor: Jocelyn Turcotte

 

Prerequisites

The prerequisites to successfully run this application are:

  • Microsoft .NET Framework Version 3.5 SP1
    • Windows 7 already includes a compatible version of the .NET Framework, so this requirement is not necessary.
    • Download from here
  • IronPython 2.6 distribution
    • If you are new to IronPython, please go over the tutorial that comes with this distribution
    • Download from here

 

Getting Started

Please make sure the folder structure in your current working directory (the DiskUse directory) is:

.\app.py

.\dialogutil.py

.\filestruct.py

.\mainWindow.xaml

.\readme.htm

.\renderers.py

.\uilogic.py

HINT: if the folder containing the IronPython distribution is not already in your PATH environment variable, please add it from the command prompt you'll be working in.  By doing this you will be able to run the "ipy" command from any arbitrary directory:

set PATH=%PATH%;%ProgramFiles%\IronPython 2.6

 

From the command prompt, navigate to the DiskUse directory and type the following:

C:\Users\username> ipy.exe app.py

Overview

  • This utility asks you to choose a base directory on your disk or network, and displays the subdirectories and files in that base directory within a TreeView widget.  When you expand a directory node of the TreeView, DiskUse calculates the sizes of all files in that directory and builds a graphical diagram that is displayed in the right pane of the main window.

  • This utility uses Windows Presentation Foundation (WPF) for user interface (UI) rendering.  It uses a simple Xaml file for UI structure and geometries to draw the diagrams on a canvas.

  • DiskUse utilizes Windows Forms for the directory selection dialog as well as to get the absolute pointer position on the screen.


 

More Details

  • The application can be started by running app.py directly or by importing app.py in a console session.  Importing app.py will run the application in a new thread allowing interactive queries. 

  • After initialization, an IronDiskUsage object is created - this object creates the main window from the Xaml file and attaches events to the window.  The two main components of the application are the file list and diagram renderer, both of which are controlled by their respective attached objects, FileListKeeper and VerticalRectanglesRenderer.
    The module filestruct interfaces with the file system and builds the DiskFile and DiskDir classes that are used by the file list and renderer to display information to the user (names and size of files and directories).

Notes

  • This utility will show file sizes and not actual "cluster space" used by them

  • To select a different base directory for DiskUse to analyze, just restart the application
